LVOCControlMechanism¶

Overview¶

An LVOCControlMechanism is an OptimizationControlMechanism that learns to regulate its ControlSignals in order to optimize the performance of the Composition to which it belongs. It implements a form of the Learned Value of Control model described in Leider et al., which learns to select the value for its control_signals (i.e., its allocation_policy) that maximzes its EVC based on a set of predictors.

Expected Value of Control (EVC) and Learned Value of Control (LVOC)¶

The Expected Value of Control (EVC) is the predicted value of executing the Composition to which the LVOCControlMechanism belongs under a given allocation_policy, as determined by its objective_function.

The LVOCControlMechanism’s learning_function learns to estimate the EVC from its current_state – comprised of feature_predictors, an allocation_policy, interactions among these, and the costs. by learning a set of prediction_weights that can predict the net_outcome of processing for experienced values of those variables. The set of prediction_weights it learns are referred to as the learned value of control (LVOC).

The LVOCControlMechanism’s primary function optimizes the EVC for the current value of its feature_predictors, by using its prediction_weights LVOCControlMechanism.prediction_weights (i.e., the LVOC) to estimate the EVC for samples of allocation_policy, and returning the one that maximizes the EVC for the current feature_predictors.

Creating an LVOCControlMechanism¶

An LVOCControlMechanism can be created in the same was as any OptimizationControlMechanism. The following arguments of its constructor are specific to the LVOCControlMechanism:

  • feature_predictors – takes the place of the standard input_states argument in the constructor for a Mechanism`, and specifies the inputs that it learns to use to determine its allocation_policy in each trial of execution. It can be specified using any of the following, singly or combined in a list:

    • {SHADOW_EXTERNAL_INPUTS: <ORIGIN Mechanism, InputState for one, or list with either or both>} – InputStates of the same shapes as those listed are created on the LVOC, and are connected to the corresponding input_CIM OutputStates by projections. The external input values that are passed through the input_CIM are used as the feature_predictors. If a Mechanism is included in the list, it refers to all of its InputStates.

    • InputState specification – this can be any form of InputState specification that resolves to an OutputState from which the InputState receives a Projection; the value of that OutputState is used as the feature. Each of these InputStates is marked as internal_only.

    Feature_predictors can also be added to an existing LVOCControlMechanism using its add_features method.

  • feature_function – specifies function of the InputState created for each item listed in feature_predictors. By default, this is the identity function, that provides the current value of the feature to the LVOCControlMechanism’s learning_function. However, other functions can be assigned, to maintain a record of past values, or integrate them over trials.

  • learning_function – specifies LearningFunction that learns to predict the EVC for a given allocation_policy from the terms specified in the prediction_terms argument.

  • prediction_terms – specifies the terms used by the learning_function and by the LVOCControlMechanism’s primary function to determine the allocation_policy that maximizes the EVC.

Structure¶

Same as an OptimizationControlMechanism, with the following exceptions.

Input¶

Like any Control Mechanisms, an LVOCControlMechanism has a primary InputState named OUTCOME that receives a Projections from the OUTCOME OutputState of its objective_mechanism. However, it also has an additional InputState for each of its feature_predictors, as described below.

Feature Predictors¶

Features_Predictors, together with the LVOCControlMechanism’s control_signals and costs are assigned to its current_state, from which its learning_function learns to predict the EVC.

Feature_Predictors can be of two types:

  • Input Feature Predictor – this is a value received as input by an ORIGIN Mechanism in the Composition.
    These are specified in the feature_predictors argument of the LVOCControlMechanism’s constructor (see Creating an LVOCControlMechanism), in a dictionary containing a SHADOW_EXTERNAL_INPUTS entry, the value of which is one or more ORIGIN Mechanisms and/or their InputStates to be shadowed. For each, a Projections is automatically created that parallels (“shadows”) the Projection from the Composition’s InputCIM to the ORIGIN Mechanism, projecting from the same OutputState of the InputCIM to the InputState of the LVOCControlMechanism assigned to that feature_predictor.
  • Output Feature Predictor – this is the value of an OutputState of some other Mechanism
    in the Composition. These too are specified in the feature_predictors argument of the LVOCControlMechanism’s constructor (see Creating an LVOCControlMechanism), and each is assigned a Projection from the specified OutputState(s) to the InputState of the LVOCControlMechanism for that feature.

The current values of the InputStates for the feature_predictors are listed in the feature_values attribute.

Functions¶

Learning Function¶

The learning_function of an LVOCControlMechanism learns how to weight its feature_predictors, allocation_policy, the interactions between these, and the costs of its control_signals, to best predict the net_outcome that results from their values. Those weights, together with the current value of its feature_predictors (contained in its feature_values attribute, are used by the LVOCControlMechanism’s primary function to estimate the EVC for those different samples of allocation_policy. By default, the learning_function is BayesGLM. However, any function can be used that accepts a 2d array, the first item of which is an array of scalar values (the prediction terms) and the second that is a scalar value (the outcome to be predicted), and returns an array with the same shape as the LVOCControlMechanism’s allocation_policy.

Note

The LVOCControlMechanism’s function is provided the feature_values and net_outcome from the previous trial to update the prediction_weights. Those are then used to estimate (and implement) the allocation_policy that is predicted to generate the greatest EVC based on the feature_values for the current trial.

Primary Function¶

The function of an LVOCControlMechanism uses the prediction_weights learned by its learning_function, together with the current feature_values, to find the allocation_policy that maximizes the estimated EVC. It does this by selecting samples of allocation_policy and evaluating each using its objective_function. The latter calls the LVOCControlMechanism’s compute_EVC method, which uses prediction_weights and current feature_values to estimate the EVC for each allocation_policy sampled. The function returns the allocation_policy that yields the greatest `EVC <LVOCControlMechanism_EVC>.

The default for function is the GradientOptimization Function, which uses gradient ascent to select samples of allocation_policy that yield progessively better values of EVC. However, any OptimizationFunction can be used in its place. A custom function can also be used, however it must meet the requirements for the function of an OptimizationControlFunction, as described here.

Execution¶

When an LVOCControlMechanism is executed, it first calls its learning_function, which uses information from the previous trial to update its prediction_weights. It then calls its function, which uses those weights to predict the allocadtion_policy that will yield the greatet EVC, and then implements that for the next trial of execution. Specifically, it executes the following steps:

  • Calls learning_function with the current_state (containing the feature_values, allocation_policy, and associated costs) for the previous trial, together with the net_outcome for that trial, and updates its prediction_weights.
  • Updates current_state with the features_values for the current trial.
  • Calls function, which uses the current feature_values and prediction_weights to determine the allocation_policy that yields the greatest EVC (see above for details).

The values in the allocation_policy returned by function are assigned as the variables of its control_signals, from which they compute their values.

compute_EVC(variable, execution_id=None)¶

Update interaction terms and then multiply by prediction_weights

Serves as objective_function for LVOCControlMechanism.

Note

If GradientOptimization is used as the LVOCControlMechanism’s function, this method (including its call to PredictionVector.compute_terms) is differentiated using autograd.grad().
